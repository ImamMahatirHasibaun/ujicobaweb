SOFTWARE DESIGN IMPLEMENTATION

    SOFTWARE DESIGN IMPLEMENTATION BY : IMAM MAHATIR HASIBUAN

    Introduction

  • This website provides an overview of the session on moving into implementation on software design, adapted from Alan Dennis, Barbara H.Wixom, and Roberta M. Roth'S "System Analysis and design" (2018)

    Learning Objectives

    By the end of this session, students should be able to:

  • Apply systems analysis and design concepts to convert a business idea into a system.
  • Create a system design by analyzing a breakthrough business idea.
  • Understand the system construction process..
  • Explain different types of tests and when to use them.
  • Develop user documentation.

    Implementation

    The implementation phase involves developing and testing the system's software, documentation, and new operating procedures. Key activities include: Writing programs

  • Writing programs
  • Managing the programming process
  • Conducting tests

    Testing

    Testing ensures the system operates correctly and meets requirements. Types of tests include:

  • Unit Tests
  • Integration Tests
  • System Tests
  • Acceptance Tests

    Documentation

    Documentation helps users and developers understand and use the system. Types of documentation include:

  • System Documentation
  • user Documentation
